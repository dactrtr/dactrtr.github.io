## Posting like a champion

Ever since I got the _iPad Pro_ I've been gradually trying to move all my habits to the small screen with the little pencil. Right now I'm testing whether, thanks to Editorial and its workflows, I've actually pulled it off.

<figure class="figimg">
	<img src="https://i.imgur.com/v7gT2rE.gif" alt="my feet">
<figcaption>
Testing the waters and how images are posted

For those who don't know, my blog and portfolio is what I call a _rat blog_ — no CMS, no hosting, just a bunch of static files and the generous gift that is **GitHub Pages**, with a little help from **Jekyll**.
Let me run through my internet combo breaker:

- domain: [namecheap](Http://www.namecheap.com) — cheap, cool domains.
- hosting: [github Pages](https://pages.github.com/) — where my site lives.
- cms (I don't have one, but this combo does the same thing):
  - [markdown](https://daringfireball.net/projects/markdown/syntax) — if you don't know this syntax, click that link.
  - [brackets](http://brackets.io/) — text editor.
  - [editorial ](http://omz-software.com/editorial/) — iOS text editor with support for creating workflows (I'll explain this more later)
  - [github desktop ](https://desktop.github.com/) — for pushing files to GitHub in the least alpha-male way possible.

> but how do you do it?

Innocent little dove.

Let's skip the part about setting up GitHub Pages with Jekyll since it's as easy as cloning the project — _ezpz_ — and go straight to the **mobile** blogging version.

### editorial

This simple plain-text editing app has something called _workflows_, and kids, **it's the best thing ever** — you can literally program actions, like proper **if**, **while** and all those **nerd programming things**. There's even a [site](https://www.editorial-workflows.com) with several GitHub workflows designed specifically for blogging. I couldn't get them working well — you had to get an API token, I got it but, nah — it's actually so simple to just upload the file from Safari, or just copy and paste. Why bother with the API.

The big advantage of **editorial** is that it lets me create snippets that fit the customizations I've made to Jekyll and the CSS, which other editors don't support. And since I'm an old sea dog used to a keyboard, typing out _figure_ tags and that kind of **HTML** by hand is impossible.

It's **3:15 am**, nuff bloggin. Besides, this is just a test post.

Test test test test test test.

## チャンピオンみたいにポストする

_iPad Pro_を買ってから、少しずつすべての習慣をちっちゃな画面とペンシルに移行しようとしています。今まさに、Editorialとそのワークフローのおかげで実現できたかどうかを試しています。

<figure class="figimg">
	<img src="https://i.imgur.com/v7gT2rE.gif" alt="私の足">
<figcaption>
水の感触を試しながら、画像のポスト方法をテスト中 
</figcaption>
</figure>

知らない人のために言うと、私のブログとポートフォリオは自分で_ラットブログ_と呼んでいるもので、CMSもホスティングもなく、静的ファイルの束と**GitHub Pages**の大きな恵み、そして**Jekyll**の少しの助けで成り立っています。
私のインターネットコンボブレイカーをおさらいしましょう：

- ドメイン: [namecheap](Http://www.namecheap.com) — 安くてかっこいいドメイン。
- ホスティング: [github Pages](https://pages.github.com/) — サイトが置いてあるところ。
- CMS（持っていないけどこの組み合わせが同じことをやってくれる）：
  - [markdown](https://daringfireball.net/projects/markdown/syntax) — この記法を知らなければクリックして。
  - [brackets](http://brackets.io/) — テキストエディタ。
  - [editorial ](http://omz-software.com/editorial/) — ワークフロー作成対応のiOSテキストエディタ（後で詳しく説明します）
  - [github desktop ](https://desktop.github.com/) — 存在する中で最も男らしくない方法でファイルをGitHubにアップロードするやつ。

> でもどうやってるの？

無邪気な子鳩さん。

GitHub PagesとJekyllの設定部分はプロジェクトをクローンするだけの_ezpz_なので飛ばして、**モバイル**ブログ版に行きましょう。

### editorial

このシンプルなプレーンテキスト編集アプリには_ワークフロー_というものがあって、みんな、**これが最高なんです** — アクションをプログラムできるんですよ、文字通り**if**、**while**、そういった**プログラミングオタク的なもの**が全部。実際に[サイト](https://www.editorial-workflows.com)があって、ブログ専用に作られたGitHub連携ワークフローがいくつかあります。うまく動かすことはできなかった — APIトークンを取得する必要があって、取得したんですが、まあいいや。実際にはSafariからファイルをアップロードするか、コピー&ペーストするだけで済むんだから、なんでAPIを使う必要があるんですか。

**editorial**の大きな利点は、私がJekyllとCSSに加えたカスタマイズに合ったスニペットを作れることで、他のエディタではそれができません。キーボードに慣れた古参者として、_figure_タグやそういった**HTML**を手打ちするのは無理なんです。

今**午前3時15分**、もうブログはいいや。それにこれはただのテストポストです。
